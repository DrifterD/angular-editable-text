Basic Usage

Try editing the titles below. An input that matches their style will be created to allow a seamless editing experience.

Validation

You can control and cancel editing by passing a function to "on-change". For more details check the README file.

Processing

The on-change function controls the return value - you can either return the value after a validation, return false to cancel the change, or return a processed value. The first example will apply censorship to bad language, and the other will always add exclamation points.

Handling Async Validation / Processing

In real world some validation or processing will involve ajax calls. To tackle this, the on-change function can also receive a promise (based on $q syntax), and once resolved with a value, the value will change accordingly. If rejected, the changes will be reverted, and in the meanwhile, a loading message will show (which is configurable).

A 2-7 seconds delay is simulated via $timeout, and there is only a 70% chance it will success

Controlling Edit Mode

Controlling and knowing when a component is in edit mode is also possible, by binding a scope variable to the edit-mode attribute. You can either control it (for example, have a button invoke the edit mode) or act upon it being edited

Transcluding Content

If there is content you wish to include inside the editable component, but isn't part of the content itself, like an icon or small message, you can put it inside the tag that has the editable-text attribute, and it will be automatically transcluded when not editing
